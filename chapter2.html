Variablen

In diesem Kapitel lernst du, wie du Werte in Variablen speichern kannst, wie deren Syntax aussieht und was bei ihnen alles zu beachten ist.

Einleitung

In den Code-Beispielen aus dem vorherhigen Kapitel "Erster Blick auf die Syntax" haben wir bisher immer nur Dinge temporär auf der Konsole ausgegeben, da wir bisher noch keine Möglichkeit hatten, Werte in irgendeiner Art und Weise zu speichern. Genau zu diesem Zweck gibt es Variablen. Doch bevor wir uns nun direkt mit deren Syntax und Funktionen beschäftigen, müssen wir erst einmal klären, wie Java allgemein im Kern nun genau funktioniert. Rekapitulieren wir zunächst noch ein paar wichtige Dinge aus dem letzten Kapitel:

  • Eine Klasse ist eine Art Bauplan für Objekte und beschreibt Operationen(=Methoden) und Eigenschaften (=Attribute), die ein Objekt ihres Typs später ausführen, bzw. besitzen kann.
  • Eine Methode kann als eine Sammlung von Anweisungen unter einem Namen angesehen werden.
  • Jedes Java Programm beginnt in der public static void main(String[] args) Methode
  • Jede Anweisung, d.h. jede Programmlogik muss in einer Klasse stehen, außerhalb von einer Klasse stehenden Anweisungen sind nicht erlaubt.

Die Grundlagen der Java Syntax

Zunächst gibt es ein paar Punkte, die man sich unbedingt im Kopf behalten sollte:

  • Case Sensitivity - Java ist case sensitive, was bedeutet, dass es möglich ist ein Objekt name und gleichzeitig ein anderes Objekt Name zu nennen.
  • Klassennamen - In Java gibt es bestimmt Namenkonventionen, denen jeder Name einer Klasse und Methode folgen sollte. So gilt für den Namen einer Klasse, dass der erste Buchstabe immer groß geschrieben wird. Für den Fall, dass ein Klassenname aus mehereren Wörtern zusammengesetzt ist, greift die CamelCase-Schreibweise und jeweils der erste Buchstabe jedes Wortes wird großgeschrieben. Beispiel: class MyFirstJavaProgram
  • Methodennamen - Auch für die Methoden gilt eine Regel: Der erste Buchstabe im Namen wird immer klein geschrieben. Der Rest des Namens folgt der CamelCase-Schreibweise. Beispiel: public void myMethodName
  • Dateinamen - Es gilt: Klassenname = Dateiname + .java. Um etwas vorzugreifen: Ist in der Datei keine Klasse mit dem public Modifizierer vorhanden, so spielt der Name der Datei keine Rolle. Beispiel: Angenommen die Klasse heißt MeinKlasse, so muss die Datei MeineKlasse.java heißen.
  • public static void main(String args[]){...} - Jedes Java Programm startet in der main() Methode, womit auch in jedem Java Programm eine solche Methode vorhanden sein muss.

Bezeichner

Um Dinge oder besser gesagt "Objekte" in Java idenzifizieren, das heißt immer eindeutig ansprechen zu können, werden für Variablen, Methoden, Klassen und Schnittstellen (Genaueres zu Schnittstellen in einem folgenden Kapitel) werden Bezeichner (Auch Identifizierer, engl: identifier genannt) vergeben. Ein Bezeichner ist ganz einfach eine Folge von Zeichen, welche Elemente aus dem Unicode-Zeichensatz sein müssen. Jedes einzelne Zeichen ist hierbei für die Identifikation wichtig (Ein Bezeichner mit 50 Zeichen muss immer mit allen 50 Zeichen korrekt angegeben werden). Jedoch gibt es auch für Bezeichner bestimmte Regeln, nach denen ihre Zeichen gewählt werden müssen. So muss ein Bezeichner ausschließlich aus Java-Buchstaben (=Alle lateinischen, griechischen und arabischen Buchstaben + noch viele weitere Zeichen aus dem Unicode-Alphabet wie €, $ und ¥) und Java-Ziffern bestehen und immer mit einem Java-Buchstaben beginnen. Obwohl die zuvorgenannten festen Regeln viele verrückte Bezeichner zulassen, sollte grundsätzlich die Programmierung mit englischen Bezeichnernamen erfolgen. Literale

Konstante Ausdrücke werden als Literal bezeichnet. Dabei wird zwischen folgenden verschiedenen Typen von Literalen unterschieden:

  • Wahrheitswerte - true und false
  • Integrale Literale für Zahlen - Zum Beispiel 432
  • Fließkommaliterale - Zum Beispiel 19.45 oder 4.321E-2
  • Zeichenliterale - Zum Beipiel 'A' oder '\n'
  • String-Literale (Für Zeichenketten) - Zum Beispiel "Matteo Mertz"
  • null - Ein besonderer Referenztyp

Kommentare

Ein fundamental wichtiger Baustein von Programmiersprachen sind Kommentare. Sie ermöglichen dem Programmierer, seine Gedanken verständlich zu formulieren, sodass auch die Nachwelt und nach einiger Zeit er selbst noch verstehen kann, was sein Code eigentlich tut. Somit dienen sie einzig und allein dem Codeverständnis des Lesers und haben keinerlei Auswirkung auf das Programm. Insgesamt gibt es drei verschiedene Möglichkeiten in Java Kommentare zu schreiben:

  • Zeilenkommentare - Beginnen mit zwei Schrägstrichen // und kommentieren das, was in der Zeile nach ihnen noch kommt aus. Somit gelten sie bis zum Ende der Zeile/dem Zeilenumbruchszeichen.
  • Blockkommentare - Kommentieren alles zwischen /* und */ aus (Wie der Name bereits vermuten lässt auch gesamte Blöcke, das heißt mehrere Zeilen). Dabei darf der Text in einem Blockkommentar selbst kein */ enthalten, da in Blockkommentaren keine Verschachtelung möglich ist.
  • Javadoc-Kommentare - Sind eine besondere Art von Blockkomentaren und kommentieren alles zwischen /** und */ aus. In ihnen wird die Methode oder deren Parameter beschrieben, aus was sich später die API-Dokumentation generieren lässt.
